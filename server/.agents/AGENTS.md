# 智能体指令与行为准则 (Agent Rules)

## 文本与文案规范

1. **客观中性**：所有输出与界面文案必须保持客观中性。
2. **拒绝夸大**：不能使用任何夸大、做作或毫无意义的装饰性词汇（例如“探索全新视界”、“安全会话已建立”、“已保护”等画蛇添足的废话）。
3. **极简直接**：文案风格保持极度简洁直接，例如用“注册”代替“创建您的账号”，用“登录”代替“安全登录”。

## 数据与环境安全规范

1. **生产意识**：对数据库和数据的任何操作必须三思。不区分“生产数据库”与“实验/测试数据库”，你正在操作的数据库即是生产数据库，没有任何测试数据库供你随意尝试。
2. **严禁破坏**：严禁执行任何具备大范围破坏性的操作（如误用 `drop_all`、清空表或覆盖已有测试数据）。任何可能改动已有表结构或数据的行为必须备份到本地一次全部数据。

## 工程规范

1. **分层结构**：应用采用 Flask 工厂模式，`app/__init__.py` 提供 `create_app()`；路由在 `app/routes/` 下按蓝图分文件（命名 `*_bp`）；业务逻辑放 `app/services/`；通用工具放 `app/utils/`；模型集中在 `app/models.py`。
2. **配置外置**：所有配置来自环境变量（`.env`），由 `app/config/settings.py` 的 `Config` 类统一加载，禁止在代码中硬编码数据库账号密码或 `SECRET_KEY`。
3. **迁移优先**：数据库表结构变更一律通过 Alembic 迁移（`flask db migrate` / `flask db upgrade`），禁止在业务代码中使用裸 `CREATE TABLE` 或 `db.create_all()`。
4. **错误不泄露**：对外接口返回结构化 JSON 错误，不得把数据库异常、堆栈等内部细节直接返回前端。
